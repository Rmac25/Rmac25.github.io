Automated IAM User Lifecycle Architecture

Joiner, Mover, and Leaver identity lifecycle design.

Project Overview

This project demonstrates a secure IAM architecture for managing the full employee lifecycle. The design supports onboarding new users, updating access when users change roles, and removing access when users leave the organization.

The architecture connects HR, IT service management, automation, identity, Microsoft 365, endpoint management, and reporting tools into one controlled workflow.

IAM Entra ID Active Directory PowerShell Microsoft 365 Intune Endpoint Central

Architecture Flow

HR System → ServiceNow/Freshservice → PowerShell Automation Server → Active Directory → Microsoft Entra ID → Microsoft 365 → Intune/Endpoint Central → Logging and Compliance Reports

Joiner Process

The Joiner process handles new employees. Once HR enters the employee record, a ticket is created and automation provisions the required accounts, groups, licenses, mailbox, Teams access, SharePoint access, and device profiles.

  • Create user account in Active Directory
  • Sync identity to Microsoft Entra ID
  • Assign role-based security groups
  • Assign Microsoft 365 license
  • Create mailbox and collaboration access
  • Assign device and software policies

Mover Process

The Mover process handles employees who change departments, managers, or job roles. Access is reviewed and updated based on the user’s new responsibilities.

  • Review current access
  • Remove unnecessary old access
  • Add new role-based access
  • Update manager and department attributes
  • Update Microsoft 365, Teams, and SharePoint permissions
  • Log changes for audit review

Leaver Process

The Leaver process handles offboarding. The goal is to quickly block access, preserve business data, recover devices, and maintain a clean audit trail.

  • Disable Active Directory account
  • Block Entra ID sign-in
  • Revoke active sessions
  • Remove group memberships
  • Convert mailbox to shared mailbox if needed
  • Transfer OneDrive access to manager
  • Retire, wipe, or recover assigned device

Security Controls

  • Role-Based Access Control
  • Least privilege access
  • Multi-Factor Authentication
  • Conditional Access policies
  • Approval workflows
  • Access reviews
  • Audit logging
  • Device compliance enforcement

Tools Used

  • Microsoft Entra ID
  • Active Directory
  • PowerShell
  • Microsoft Graph
  • Microsoft 365 Admin Center
  • Exchange Online
  • SharePoint Online
  • Microsoft Intune
  • ManageEngine Endpoint Central
  • ServiceNow or Freshservice
