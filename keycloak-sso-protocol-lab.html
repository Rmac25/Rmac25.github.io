Federated authentication

Problem

SSO breaks when protocol details are misunderstood.

Federated authentication depends on correct client settings, redirect URIs, claims, scopes, and token behavior. A small configuration mistake can cause login loops, missing roles, or blocked application access.

Solution

A lab that follows the authentication path end to end.

This project uses Keycloak as the identity provider to model how users authenticate, how applications redirect to the provider, and how identity data returns to the application.

Protocol flow

How SSO works in the lab

01

Request

The user opens the application. The app redirects the user to Keycloak because authentication is required.

02

Authenticate

Keycloak validates the user, applies realm/client rules, and prepares the assertion or token response.

03

Return

The browser returns to the approved redirect URI, and the app consumes the identity data for access decisions.

Configuration matrix

SSO design decisions

Realm

Creates an isolated identity space for users, clients, roles, groups, and authentication behavior.

Client

Represents the application that relies on Keycloak for authentication and token/assertion handling.

Redirect URI

Defines the approved callback URL so Keycloak only returns users to trusted application endpoints.

Claims

Passes user attributes, roles, and groups so applications can make authorization decisions.

  7. Break one setting intentionally and document the troubleshooting signal.

Troubleshooting evidence

What I would check when SSO fails

The lab connects common SSO symptoms to likely IAM causes so an interviewer can see the troubleshooting mindset, not just a finished setup.

SSO troubleshooting checklist
- Confirm client ID and access type are correct
- Confirm redirect URI matches the application callback
- Confirm user exists and is enabled in the realm
- Confirm required roles, groups, scopes, or mappers exist
- Review token claims for subject, email, roles, and groups
- Check for invalid client, redirect mismatch, and expired session errors
- Compare expected IdP/SP behavior against the actual login flow
