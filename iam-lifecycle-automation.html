Project case study

IAM Lifecycle Automation

A PowerShell-based identity workflow for onboarding, access review, offboarding, and audit evidence across Microsoft Entra ID and Active Directory environments.

Joiner / mover / leaver lifecycle
Security
Least privilege, audit logging, safe testing

Problem

Manual identity work creates risk.

Onboarding, access reviews, and offboarding often depend on repeated administrative steps. If those steps are inconsistent, access can be over-provisioned, evidence can be incomplete, and offboarding can miss active sessions or privileged assignments.

Solution

A repeatable IAM workflow.

This project standardizes IAM actions into one controlled PowerShell workflow with provider selection, parameter validation, audit logging, Microsoft Graph support, Active Directory support, and safe execution through `-WhatIf` / `-Confirm`.

Workflow

What the script demonstrates

01

Onboard

Create or update an identity, assign baseline attributes, set manager details, and add requested group memberships.

02

Review

Pull identity details and group memberships, print reviewer-friendly output, and optionally export JSON evidence.

03

Offboard

Disable the account, revoke Graph sign-in sessions, remove requested groups, app role assignments, directory role memberships, and supported authentication methods.

Demo commands

How the project is used

Safe onboarding test

.\IAM-Protocol.ps1 `
  -Action Onboard `
  -Provider MicrosoftGraph `
  -UserPrincipalName "jane.doe@example.com" `
  -DisplayName "Jane Doe" `
  -Department "Information Technology" `
  -JobTitle "Security Analyst" `
  -Manager "manager@example.com" `
  -RequestedGroups "IAM-Users","VPN-Access" `
  -TicketId "CHG-1001" `
  -WhatIf

Access review evidence

.\IAM-Protocol.ps1 `
  -Action Review `
  -Provider MicrosoftGraph `
  -UserPrincipalName "jane.doe@example.com" `
  -TicketId "REVIEW-1001" `
  -ReviewOutputPath ".\review-jane-doe.json"

Engineering decisions

Security controls built into the project

Provider selection

Supports Microsoft Graph as the default provider while keeping Active Directory available for hybrid or legacy environments.

Safe execution

Uses PowerShell `ShouldProcess`, `-WhatIf`, and `-Confirm` so risky changes can be reviewed before execution.

Audit trail

Writes JSON-formatted log events with ticket ID, provider, user, action, level, timestamp, and message.

Offboarding depth

Graph offboarding includes account disablement, sign-in session revocation, group removal, app role cleanup, directory role cleanup, and authentication method cleanup.
