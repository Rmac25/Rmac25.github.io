Project case study

Active Directory Access Review Lab

A hands-on IAM lab for reviewing Active Directory users, group memberships, stale accounts, privileged access, and least-privilege cleanup evidence.

Download Runbook Back to Portfolio
Platform
Active Directory
Focus
Access review and group cleanup
Tools
ADUC, PowerShell, RSAT
Security Goal
Least privilege and audit readiness

Problem

Access grows stale without regular review.

Active Directory environments can accumulate disabled users, stale accounts, outdated group memberships, and privileged access that no longer matches business need.

Solution

A repeatable AD access review process.

This lab demonstrates how to inventory users and groups, compare access against business need, document findings, and test cleanup actions safely before making changes.

Review workflow

How the AD lab is structured

01

Inventory

List users, disabled accounts, stale accounts, and key security groups using ADUC and PowerShell.

02

Validate

Compare group memberships against an access matrix, ticket, department, or business justification.

03

Remediate

Document recommended removals, test with `-WhatIf`, and retain evidence for audit or access certification.

Design matrix

Active Directory review decisions

Users

Review enabled, disabled, stale, and department-based user records for account hygiene.

Groups

Validate baseline, application, VPN, and privileged group memberships against business need.

Evidence

Export users and group memberships to CSV or JSON for ticket attachment and audit review.

Cleanup

Use `-WhatIf`, approvals, and change tracking before removing access or disabling accounts.

Practice steps

Hands-on lab path

Download Full Runbook
  1. Create test users for active, disabled, and stale account scenarios.
  2. Create security groups for baseline access, VPN access, and admin access.
  3. Add users to groups based on a simple access matrix.
  4. Use PowerShell to list users, account state, and last logon details.
  5. Export group memberships and compare them to business need.
  6. Identify excessive access and test cleanup commands with `-WhatIf`.
  7. Document findings as audit evidence for an access review.

Evidence commands

PowerShell used for review

The project shows practical AD review commands that can produce evidence for stale account checks, group review, and cleanup validation.

Get-ADUser -Filter * -Properties Enabled,LastLogonDate,Department |
  Select-Object Name,UserPrincipalName,Enabled,LastLogonDate,Department

Get-ADGroupMember -Identity "VPN-Access" |
  Select-Object Name,SamAccountName,ObjectClass

Remove-ADGroupMember -Identity "VPN-Access" `
  -Members "jane.doe" `
  -WhatIf
