Active Directory Cleanup

A hands-on IAM cleanup project for finding stale accounts, reviewing disabled users, validating group memberships, and preparing least-privilege remediation evidence.

Problem

Directory access gets messy over time.

Active Directory environments can accumulate stale accounts, disabled users with lingering access, outdated group memberships, and privileged access that no longer matches business need.

Solution

A repeatable cleanup review process.

This project demonstrates how to inventory accounts, classify cleanup candidates, review group access, export evidence, and test remediation safely before making changes.

Cleanup workflow

How the AD cleanup project is structured

01

Inventory

Export enabled, disabled, stale, and no-logon accounts using PowerShell and Active Directory attributes.

02

Validate

Review privileged groups and business-access groups against business need, ticket history, or access matrices.

03

Remediate

Document cleanup candidates, test removals with `-WhatIf`, and retain evidence for audit or access certification.

Design matrix

Active Directory cleanup decisions

Users

Classify enabled, disabled, stale, and no-logon accounts before recommending cleanup.

Groups

Validate baseline, application, VPN, and privileged group memberships against business need.

Evidence

Export account review results and group membership findings for ticket attachment and audit review.

Cleanup

Use approvals, change tracking, and `-WhatIf` before removing group access or disabling accounts.

Practice steps

Hands-on lab path

Download Full Runbook PowerShell script
  1. Create test users for active, disabled, stale, and no-logon account scenarios.
  2. Create security groups for baseline access, VPN access, and admin access.
  3. Add users to groups based on a simple access matrix.
  4. Run the cleanup review script to export account status evidence.
  5. Review privileged and business-access group memberships.
  6. Identify excessive access and test cleanup commands with `-WhatIf`.
  7. Document findings as audit evidence for an access review.

Evidence commands

PowerShell used for cleanup review

The project includes a safe review script and practical AD commands that produce evidence for stale account checks, group review, and cleanup validation.

.\AD-Cleanup-Review.ps1 `
  -StaleDays 90 `
  -ReviewGroups "Domain Admins","VPN-Access" `
  -ExportPath ".\ad-cleanup-review.csv"

Get-ADGroupMember -Identity "VPN-Access" |
  Select-Object Name,SamAccountName,ObjectClass

Remove-ADGroupMember -Identity "VPN-Access" `
  -Members "jane.doe" `
  -WhatIf
